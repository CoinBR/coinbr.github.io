Task: Analisar Viabilidade da História de Usuário
As Histórias de Usuário de um Administrador será analisada
Purpose
Poupar tempo e recursos que seriam desperdiçados em uma funcionalidade inviável
Relationships
RolesPrimary Performer:
  • Gerente
Additional Performers:
  • Desenvolvedores
InputsMandatory:
  • História de Usuário
    		Optional:
    • None
    Outputs
    • História de Usuário
      Main Description

      Usuários podem requisitar algo funcionalidades inviáveis, tais como:

      "Eu, como Administrador, gostaria de saber quantos carros passaram em frente ao nosso estabelecimento durante o dia"


      Tal funcionalidade iria requerer um esforço sem tamanho por parte dos Desenvolvedores, e não entregaria tanto valor para o negócio. É nesse momento que o gerente deve encontrar pessoalmente tanto com o Usuário que solicitou a funcionalidade, como com os demais stakeholders, e tentar encontrar uma solução alternativa para aquele anseio, como pegar anualmente os dados que a prefeitura libera sobre o tráfego de carros naquela rua.

      Em casos de Histórias de Usuários coerentes, que deverão ser maioria, o Gerente irá encaminhá-las para os Desenvolvedores implementarem
